# Future work — deliberate scope boundaries as of v0.1.0 (final review)

Each row is a design-spec or audit promise intentionally not in v0.1.0, with the one-line
rationale recorded by the final review (docs/status/final-review.md §D).

| Item | Spec/audit ref | Rationale |
|---|---|---|
| `natex validate` / `natex estimate` / `natex report` subcommands | spec §3 (cli.py line) | Folded by design: validation + estimation run inside `discover` (`src/natex/discover.py:266-297`) and reporting is the bundle + `paper`/`brief`; the real 9-command surface is documented in README/AGENTS and the fold-in recorded in `docs/status/phase-report-paper.md:129`. |
| `legacy/` annotated copy of the original Herlands `src/` | spec §3 | Permanently dropped — never commit third-party code (global constraint); the legacy bug list is cross-referenced in `docs/method_cards/lord3.md:57-63` + `docs/math_audit_final.md`, each bug class pinned by a regression test (final-review §A items 20-24). |
| mypy (loose) step in CI | spec §3 ci.yml comment | CI gates on ruff + pytest across 3.11-3.14; a loose mypy step was never wired and no shipped doc claims typechecking — add later as a non-blocking job. |
| rdrobust bridge estimator | spec §3 estimate/, §10 risk 3 | Own frozen-side local 2SLS + Wald ship as the estimators (`src/natex/estimate/`); wrapping the rdrobust port would add a non-core dependency for a cross-check natex already gets from backtests. |
| rddensity bridge for the density test | spec §3 validate/, §10 risk 3, audit §2 item 6 (F-A2) | Own binned-Poisson McCrary-style analogue on the frozen signed distance ships, documented falsification-only (`src/natex/validate/density.py:1-5`); the local-polynomial rddensity port stays a deliberate non-core deferral. |
| Honest-split auto-wiring in the `discover`/`study` pipelines | spec §4 (`honest=True`), audit §2 item 6 (F-A2) | `honest_split` ships (`src/natex/validate/honest.py`) and is wired into the IV pipeline; the RDD/DiD pipelines run density/placebo on the full discovery sample with the caveat documented — auto-wiring is the follow-up. |
| Google Docs API export | spec §7.2 | Intentionally manual: render markdown, then gws-docs / "Open with Google Docs" (README:290-294; natex-write-paper SKILL §4) — no Google API dependency in core, agent mode covers the automated route. |
| `--diagrams paperbanana` CLI flag | spec §7.3 | Diagram generation ships as the Python adapter `report/paperbanana.py` + `paperbanana` extra (README:252-258); a CLI flag on `natex paper` is deferred until the adapter sees real use. |
| PyPI publish | spec §2, §9 phase 8 | v0.1.0 is a GitHub-only release; the dist name `natex-discovery` is reserved (plain `natex` unavailable) and publishing awaits the user's naming decision (`docs/release_notes/v0.1.0.md:97-99`). |
| Staggered-adoption group-time ATT backend (Callaway-Sant'Anna) | spec §1 non-goals, audit §3 adopted 9 (F-B3) | Exactly the promised boundary: the pluggable `DiDEstimatorBackend` protocol ships with `mean_gap` registered (`src/natex/did/effects.py:316-345`); the group-time ATT backend is the documented future backend (`docs/method_cards/suddds.md:185-186`). |
| Event-study estimator (leads/lags coefficients with CIs) | spec §3 estimate/ ("incl. event-study plot data") | Per-period gap data ships (`pretrend_plot` inputs, `DonorSelectionResult.effect_by_time`); a coefficient-based event-study estimator belongs with the staggered-adoption backend above. |
| Mid-run `propose_forcing_variables` / `propose_candidate_events` hooks | spec §6c (documented deviation) | Folded into the Stage-0 `search_plan` task (`docs/method_cards/llm_analyst.md:43,202`); separate mid-run refinement tasks wait for a use case that needs re-ranking inside a running scan. |
| Sandboxed LLM-written pandas escape hatch | spec §6a.3 | Prep is strictly declarative — the library executes a versioned `PrepPlan`, the LLM never touches data; the sandboxed-snippet hatch is documented out of scope (`docs/method_cards/llm_analyst.md:200`). |
| ANN neighbor-search backend (opt-in) | spec §6d, audit §3 rejected 4 (downgrade) | All neighbor queries are exact `scipy.spatial.cKDTree`; the audit permits ANN only when all replicas share the approximation, so an opt-in backend waits until dataset scale demands it. |
| Parallelism across neighborhoods | spec §6d (F-E2) | The scan is serial and bitwise-deterministic; the shipped speedups (vectorized LLR, geometry caching across replicas, coarse-to-fine, homogeneous pruning, k-prefix reuse) carry the scale story — process parallelism must not break seed-identical output. |
| Dedicated LLR-ranking figure | spec §7.1 (F-E3) | Ranking is delivered by `discovery_scatter`'s LLR color map plus the ranked top-20 tables in `results.json`/the paper; a fifth standalone figure is cosmetic. |
| Multi-k joint scan calibration | spec §4 (`k` parameter), audit §3 adopted 4 | Each run scans one user-chosen `k` (default 50) with the null calibrated at that same `k`; `ScanGeometry.shrink` already gives exact k-prefix reuse, so a joint max-over-k statistic with a shared null is a cheap future extension — today multi-k means separate seeded runs. |
| Egger-Koethenbuerger strict decoy-free donor support | spec §8/§10 stretch goal | The core >=2-statutory-thresholds backtest passes with zero tuning (`tests/backtests/test_egger_koethenbuerger.py`); the strict decoy-free-support variant is the documented stretch failure, pinned as `xfail` (`tests/backtests/test_prop99_donors.py:199-204`) exactly as spec §10 prescribes. |
| GLR common-variance kernel as a `lord3_scan` statistic option | audit §3 adopted 1 (F-B2) | The adopted unknown-variance GLR is implemented and tested (`src/natex/scan/statistics.py:39`, `tests/test_statistics_normal.py:40`) but the wired Normal scan kernel is the thesis/KDD-parity heteroskedastic plug-in; exposing `statistic="glr"` from `lord3_scan`/the CLI restores the adopted default-vs-option polarity. |
| Chi-square measurement-model opt-out (`smooth_noise_stage1=False`) | audit §2 item 9 (F-A3) | The non-default opt-out feeds raw HC1 SE² to the GPs as known per-point noise — kept only as a documented comparison mode; removing it or renaming it to a diagnostics-only flag is the follow-up. |
| Poisson treatment model for count treatments | audit §2 item 19 (F-B4) | `model="auto"` maps binary theta to Bernoulli and everything else to Normal, so count treatments get the Normal-residual model; the Poisson variant is documented future work (`docs/method_cards/suddds.md:74-75,179-180`). |
| Per-discovery step-down selective inference (labeled approximate) | audit §3 adopted 5 (F-B5) | Holm step-down ships inside the validation batteries; the discovery-level guarantees are the max-LLR fitted-null calibration plus the honest split (the decision's PRIMARY guarantee, implemented) — no shipped doc claims a step-down guarantee. |
| Overlap-aware IF/bootstrap-covariance aggregation | audit §3 adopted 6 (F-B6) | The decision's own simple default — disjointness by construction via the VKNN repair — is implemented (`src/natex/dee/vknn.py:75-141`); influence-function aggregation was considered and deferred (`docs/method_cards/dee.md:159-161`). |
| `lord3_scan` dead `rng` parameter | API sweep B1 (F-C3) | The scan body never consumes its `rng` — the scan is fully deterministic; either drop the parameter or document it as accepted-for-interface-uniformity in a docstring. No correctness impact. |
| Loud ValueError for negative `--m-prime` | API sweep B6 (F-C4) | `_ordered_candidates` silently clamps a negative `m_prime` to 0 (`src/natex/dee/vknn.py:69`), yielding zero candidates instead of raising like the module's other guards; fail-visible downstream but a silent coercion of a user budget. |
| `tau`/`se`/`ci` naming adjudication for `DonorSelectionResult` | API sweep B3 (F-C5) | The SC vertical deliberately uses domain-standard `att_post`/`effect_by_time` naming with RMSPE-ratio placebo inference (no `se`/`ci`), consistent across CLI/skills/method card; adjudicate against the tau/se/ci convention rather than rename mechanically. |
| Silence sklearn `OptimizeWarning: iprint` stderr noise | docs execution §C row 3 (F-D5) | Not a natex defect (sklearn's LogisticRegression passes `iprint` to a scipy L-BFGS that dropped it) and stdout matches the documented console blocks byte-for-byte; candidate cosmetic fix is a targeted warnings filter around the logistic fits or a dependency floor note. |
| Guidance-log `seq` <-> request-filename reconciliation | docs execution §C row 21 (F-D6) | After an invalid agent answer falls back to NullBackend, the log `seq` and the request-file counter diverge permanently; no documented claim is violated (AGENTS.md ties sequencing to request files), but logging the request filename in each entry would make reproducibility audits unambiguous. |
| Spec §4 convenience API surface | spec §4 (F-E1) | The spec draft's `DatasetSpec.from_csv`, `seed=` kwargs, `study(...).report`, and `result.discoveries`/`.to_report()` were superseded by the shipped surface (explicit `rng=` per the one-Generator rule, `Dataset.from_csv`, `DiscoverReport.best()`, bundle + `natex paper`); every shipped doc teaches the real forms — convenience aliases only if users ask for them. |
