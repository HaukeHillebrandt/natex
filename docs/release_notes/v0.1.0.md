# natex v0.1.0 — automated natural-experiment discovery

First tagged release. natex takes a tabular CSV end to end: an optional LLM analyst pass
profiles the data and drafts a search plan (`natex study`), LoRD3 scans for
regression-discontinuity candidates and SuDDDS for difference-in-differences designs
without ever reading the outcome, every candidate faces a validation battery (fitted-null
Monte Carlo, placebo, and density tests), effects are estimated honestly (frozen
side-indicator 2SLS, DEE-debiased CATEs, BCCH-Lasso instrument search, ADH
synthetic-control donors), and the results bundle renders standard figures plus an
AI-draft paper with a mandatory review banner. Every algorithm is implemented as
corrected against the mathematical audit of the source papers and their released code —
not as printed. Determinism is a design rule: one seeded `numpy.random.Generator` flows
through every stochastic call (same seed, same result), and failed computations return
`NaN`, never `0.0`.

## Methods

| Method | What it does | Correction vs the papers |
|--------|--------------|--------------------------|
| LoRD3 (RDD scan) | Log-likelihood-ratio scan over local KNN neighborhoods for treatment-assignment discontinuities, with a coarse-to-fine gate for large data | Honest fitted-null +1-rank Monte Carlo p-values (a parametric bootstrap, stated as such); `Bernoulli(p̂)` null replicas (the legacy thresholded-Gaussian draw has the wrong success probability); own-k neighborhood variance with a data-scaled floor (legacy indexed reverse neighbors with an absolute `1e-6` floor); boundary likelihood suprema for sharp splits instead of `NA`; explicit hyperplane tie convention (signed distance ≥ 0 ⇒ group 1) |
| SuDDDS (DiD scan) | Scans unit×time panels for (group, adoption-period) treatment shifts against a fitted background model | Sigma-squared precision weights (the thesis prints `sigma_i`; Eq 6.9 only follows with squared weights); single-Δ statistic profiles the per-profile means out under both hypotheses and restricts to the window (the printed statistic treats them as known); corrected synthetic DGP (Eqs 6.24–6.25 are dimensionally invalid as printed) |
| DEE (debiased CATE) | Debiases an observational CATE learner with discovered-experiment estimates via a Voronoi–KNN repair and a mixture posterior | Mixture posterior keeps between-model covariance (the released code drew an independent model indicator per point, collapsing aggregate CIs); the paper's identification-error bound is never computed — Lemma 2's step is false; the radius diagnostic is reported with the repaired √2/fixed-shape caveat |
| IV search | BCCH plug-in Lasso instrument search plus honest 2SLS with J and Anderson–Rubin inference | Honest discovery/estimation split: selection sees only the discovery half, so estimation-half J/AR/Wald p-values keep their nominal distributions with no post-selection correction; `honest=False` carries an explicit optimism caveat; AR confidence sets report their kind (interval/unbounded/disjoint) instead of silently coercing |
| SC donors (synthetic control) | ADH donor-pool selection on the simplex plus in-space placebo inference | In-space placebo test is the corrected two-sided studentized version; empty unit×time cells are `NaN`, never `0` |
| Effect estimation (RDD) | Local 2SLS at each discovered cutoff | Frozen side-indicator instrument with HC1 errors — the papers' group instrument (Eq 5.14) is inconsistent as printed; placebos are local intercept-continuity contrasts (side-mean contrasts mechanically reject valid RDDs); density falsification runs on the signed distance along the frozen discovered normal |

Full derivations: the [math audit](https://github.com/HaukeHillebrandt/natex/blob/v0.1.0/docs/math_audit_final.md)
is the governing document; per-method cards live in
[docs/method_cards](https://github.com/HaukeHillebrandt/natex/blob/v0.1.0/docs/method_cards).

## Backtests

Real-data backtests (run with `-m backtest` and `NATEX_DATA` set; datasets are never
committed). natex is never told the answer — it must rediscover it:

| Dataset | Design | Result |
|---------|--------|--------|
| `test_score_2012` | Sharp RDD | Pretest-215 cutoff recovered; known τ ≈ 10 inside the 2SLS CI with a strong first stage |
| `academic_probation` | Fuzzy RDD | `dist_from_cut` ranked #1 of 4 candidate forcing variables; runs through the coarse-to-fine scan |
| `ed_visits` | Fuzzy RDD | Insurance-loss cutoffs at ages 19 and 23 recovered among the top clusters |
| `inpatient_visits` | Fuzzy RDD | Age-23 cutoff recovered on only 73 aggregated cells (small-n robustness) |
| `egger_koethenbuerger` | Multi-cutoff RDD | ≥ 2 statutory council-size population thresholds discovered on `log_pop` (4 of 5 observed) |
| `prop99` | DiD + synthetic control | (California, 1989) recovered with Table 6.1-consistent signs; SC donors recover the ADH pool (weight 0.955 on ADH's five donors, ATT −19.5) |

## LLM analyst + guidance

`natex study` runs the Stage-0 analyst pass: profile the CSV, draft a prep plan and a
search plan, and answer six fixed guidance tasks. Four interchangeable backends:
`null` (deterministic, never vetoes — the default; the whole pipeline runs with no LLM at
all), `agent` (a file protocol — natex writes each question as a JSON request under
`OUT/guidance/requests/` and polls for your response, so any agent or human can serve
it), and `anthropic` / `gemini` API backends behind the optional `[llm]` extra. Guidance
is **advisory only**: it never alters a statistic — results are bitwise identical with
and without it — and every request and response is logged to `out/guidance_log.jsonl`.

## Install

Requires Python >= 3.11. Not on PyPI — install from GitHub:

```bash
uv add git+https://github.com/HaukeHillebrandt/natex
```

From source:

```bash
git clone https://github.com/HaukeHillebrandt/natex
cd natex
uv sync --extra dev
```

Optional extras (everything runs on core numpy/scipy/pandas/scikit-learn/typer/pydantic
without them): `plot` (figures), `report` (jinja2 paper templates), `paperbanana`
(methodology diagrams), `ml` (econml causal forest), `gp` (GPyTorch/botorch backend),
`llm` (Anthropic/Gemini guidance backends) — e.g.
`uv add "natex-discovery[plot,report,llm] @ git+https://github.com/HaukeHillebrandt/natex"`.
